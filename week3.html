Git & HTML 版本控制

学习 Git 工作流，并用 HTML/Gitee Pages 建立项目展示意识。

第三周：Git一、什么是Git？Git是目前全球最主流的分布式版本控制系统（Distributed Version Control System, DVCS）。它由 Linux 之父 Linus Torvalds 开发，最初是为了管理庞大的 Linux 内核源码而设计，现已成为软件工程和创客项目管理的行业标准。Git是一个开源的分布式版本控制系统，能够在本地完整保存项目的修订历史，通过提交（commit）、分支（branch）、合并（merge）等机制，实现对代码及文档版本的精细管理，并可与 Gitee、GitHub 等远程仓库配合完成多人协同开发与备份。它允许每个开发者在本地拥有完整的代码仓库副本（包括完整的历史记录），从而实现离线操作和高速的版本切换。

二、Git工作链：

1. 工作区（Working Directory）就是你电脑里的真实文件夹，里面放着你正在写的：Arduino / ESP32 代码3D 模型文件HTML / CSS / JSREADME、实验记录等特点：这里的文件还没被 Git 正式“记账”。暂存区（Staging Area）使用命令：git add 文件名git add .作用：告诉 Git：“这些改动，我打算在下一次提交中一起保存。

”特点：相当于“打包准备区”，方便你把相关的修改整理成一个有意义的版本。本地仓库（Local Repository）使用命令：git commit -m "描述这次做了什么"作用：Git 会把暂存区的内容永久记录为一个版本快照，生成唯一 ID。特点：保存在你本机 .git目录中；可以随时回退、对比、查看历史；

不需要联网4.远程仓库（Remote Repository）常见地址：Gitee（国内常用）GitHub / GitLab推送命令：git push origin main拉取命令：git pull origin main作用：把本地提交同步到云端；把队友的更新同步到本地；

作为项目成果的公开展示与备份完整工作链：在工作区写代码 → 用 git add选入暂存区 → 用 git commit保存到本地仓库 → 用 git push同步到 Gitee 远程仓库，实现版本管理与团队协作。

三、第三周总结与体会：

本次实训我了解并安装了Git工具，在Gitee上注册并新建仓库，通过git clone将远程仓库拉取到本地，使用文本编辑器编写一个包含基本HTML标签（如标题、段落、图片）的index.html文件，再通过git add .、git commit -m ""、git push将代码提交至Gitee远端；最后在仓库中开启Gitee Pages服务，获得一个公网可访问的静态网页链接，实现创意项目文档或作品集的在线呈现。

这一过程不仅让我初步掌握分布式版本控制与静态网页托管的基本流程，也体会到开源协作平台在创客项目文档化、可视化传播中的实际价值。

第三周核心代码片段
git clone <repository-url>
git add .
git commit -m "update weekly project log"
git push origin main
