嵌入式计算：双向定点伺服控制系统

用微动开关、舵机和 LED 反馈构成一个可观察的交互系统。

第六周：嵌入式计算嵌入式计算的概念：嵌入式计算（Embedded Computing）是指将专用计算系统（微处理器/微控制器 + 存储器 + I/O 接口）嵌入到特定仪器设备或非计算机设备中，通过预置的软件程序实时采集外部信号、进行逻辑或算法处理，并直接控制执行机构完成既定功能，通常具有专用性强、资源受限（算力/功耗/体积）、高实时性与高可靠性要求的特点。它不同于通用 PC 的通用计算模式，而是让“计算能力”隐匿于家电、汽车电控、工业机床、智能穿戴及物联网节点之中，使物理设备具备感知、决策与响应能力。WiFi无线网络节点连接配置的核心代码：ESP32 WiFi无线网络节点连接配置的核心代码，实现开发板主动接入局域网并获取云端IP的数据连接能力。#include <WiFi.h>// 替换为你的路由器SSID和密码const char* ssid = "WiFi 名称";const char* password = "WiFi密码";void setup() { Serial.begin(115200); 四、第六周总结与体会：

通过本次基于ESP32-S2的双向定点伺服控制组件的实践，我深刻体会到了嵌入式系统中“指令-执行-反馈”闭环逻辑在实际硬件交互中的重要性。从微动开关的物理触发到舵机的精确角度切换，再到LED的同步视觉反馈，每一个环节都需要软硬件的高度协同与严谨调试。在排查公共接地路径与LED极性的过程中，我意识到硬件层面的细节往往决定了系统的稳定性与可靠性，而代码中的防抖处理与阻塞等待机制也让我更加理解了实际工程中信号噪声与用户误触带来的挑战。

这次项目不仅让我掌握了舵机控制与开关量输入的基本方法，更让我建立起从代码逻辑到物理执行的完整工程思维，为后续更复杂的物联网与自动化系统开发打下了坚实基础。

第六周核心代码片段
#include <ESP32Servo.h>

Servo myServo;
const int servoPin = 5;

// 定义开关引脚
const int btnLeft = 1;
const int btnRight = 2;

// 定义 LED 引脚
const int ledLeft = 3;
const int ledRight = 4;

void setup() {
  Serial.begin(115200);
  
  myServo.attach(servoPin, 500, 2400);
  
  // 设置按键为输入上拉模式（未按下时为高电平，按下为低电平）
  pinMode(btnLeft, INPUT_PULLUP);
  pinMode(btnRight, INPUT_PULLUP);
  
  pinMode(ledLeft, OUTPUT);
  pinMode(ledRight, OUTPUT);
  
  // 初始位置
  myServo.write(90); 
}

// 闪烁三次的函数
void blinkLED(int pin) {
  for (int i = 0; i < 3; i++) {
    digitalWrite(pin, HIGH);
    delay(150);
    digitalWrite(pin, LOW);
    delay(150);
  }
}

void loop() {
  // 检测左转开关 (按下时读取到 LOW)
  if (digitalRead(btnLeft) == LOW) {
    Serial.println("向左转...");
    myServo.write(0);      // 舵机转到 0 度
    blinkLED(ledLeft);     // 对应 LED 闪烁三下
    while(digitalRead(btnLeft) == LOW); // 等待松开，防止重复触发
  }

  // 检测右转开关
  if (digitalRead(btnRight) == LOW) {
    Serial.println("向右转...");
    myServo.write(180);    // 舵机转到 180 度
    blinkLED(ledRight);    // 对应 LED 闪烁三下
    while(digitalRead(btnRight) == LOW); // 等待松开
  }
  
  delay(10);
}
